## 正式目录

### 第 1 章 概念、边界与适用场景

- 章节职责: 解释深度研究是什么、不是什么、何时值得启动。
- 必须覆盖: 定义、与普通搜索的区别、四原则、适用场景、典型产出。
- 禁止越界: 不提前细讲第 2-5 章的方法细则。

深度研究是一种面向复杂问题的循证研究活动，适用于信息不完整、问题有争议、后果较大的情境。它的目标是在现有证据条件下构造一条可检查、可追溯、可复核的认知路径，让结论、依据和限制同时被看见。

要先划清边界。深度研究不等于普通搜索加量。普通搜索通常追求快速命中一个可用答案，适合定义查询、事实定位、资料入口查找等低歧义任务。深度研究则面对复杂、模糊、开放或争议性问题，要求研究者或无状态 LLM 同时回答“是什么”“为什么这样判断”“依据来自哪里”“还有什么没有被证实”。如果一个问题只需要查日期、网址或术语，直接搜索通常更高效；如果需要跨来源比较、处理冲突信息、评估结论可靠性，才值得启动深度研究。

深度研究建立在四个原则之上。

第一，循证（evidence-based）。所有关键判断都应尽可能落在可追溯证据上，而不是经验、语气、权威感或单一转述。这里的重点不是“有资料就算证据”，而是证据来源能被指出、内容能被复查、使用方式能被解释。

第二，迭代（iterative）。深度研究不是一次检索后立刻收束，而是随着新证据进入不断修正理解边界。一个阶段性的判断只代表“当前最好解释”，后续证据可以增强它，也可以推翻它。研究过程本质上是连续更新，而不是一次定论。

第三，可证伪（falsifiable）。好的研究不是只会搜集支持自身观点的材料，而是主动保留“我可能错了”的空间。一个判断如果没有潜在反证入口，就很容易滑向立场表达，而不是研究。深度研究要求问题和阶段性结论都能接受挑战。

第四，多源交叉验证（multi-source verification）。单一来源可以提供线索，但通常不足以独立支撑高置信结论。深度研究强调从不同类型、不同立场、不同链路的来源进行比对，观察它们是相互印证、部分冲突，还是根本不兼容。结论的可信度，不取决于某一条信息是否响亮，而取决于证据链是否稳固。

基于这四个原则，深度研究适用于几类典型场景。其一，问题本身高度复杂，例如一个现象可能同时受到时间、主体、机制、约束条件等多因素影响，单一资料无法说明全貌。其二，问题表述模糊，原始提问里混有多个子问题，需要先厘清研究边界。其三，信息彼此冲突，不同来源给出互相矛盾的说法，需要判断谁更可靠、矛盾发生在什么层面。其四，问题具有较高利害关系，例如影响决策、投入、风险控制、对外表达或公共判断，此时“差不多正确”并不够。其五，研究对象处于快速变化环境中，需要同时处理时间版本与当前有效性。

相应地，也要知道什么时候不该用深度研究。如果任务只是获取一个基础事实、确认一个公开定义、寻找一个已知资源入口，或者时效要求极高且只需要临时参考，那么直接搜索、快速浏览或普通资料整理更合适。深度研究有成本，只有当问题复杂度、不确定性或风险水平超过普通搜索的适用范围时，它才值得被启动。

深度研究的典型产出不是一段笼统结论，而是一组结构化交付物。最核心的是证据链：关键事实来自哪些来源，它们如何互相支持或互相限制。其次是推理链：从事实到中间判断，再到最终结论，逻辑是怎样成立的。第三是置信度判断：当前结论处于高、中、低哪一档可信水平，以及为什么只能到这个水平。第四是残余不确定：还有哪些缺口尚未补齐，哪些部分只能暂时保留条件性判断。第五是后续建议：如果还要继续研究，下一步最值得验证的方向是什么。

因此，可以把深度研究理解为一种面向不确定世界的认知工作方式，用来在复杂情境下保持判断的可解释性（explainability）、可复核性（reviewability）与可迁移性（transferability）。对人类读者来说，第 1 章的任务是建立边界意识：什么时候该进入深度研究，什么时候不该。对无状态 LLM 来说，第 1 章的任务是建立最小工作共识：目标不是直接给答案，而是交付带证据、带限制、带确定度的研究结果。

### 第 2 章 问题拆解与研究设计

- 章节职责: 把模糊问题转成可执行、可检验、可证伪的研究计划。
- 必须覆盖: 问题拆解、假设驱动、研究路径、信息需求、时效性分支。
- 禁止越界: 不在本章展开来源分级、交叉验证和输出模板。

<!-- chapter:02:start -->
第 2 章的任务，是把一个“现在还说不清楚的问题”转成一个“已经可以开始执行的研究计划”。对人类读者来说，这一章提供的是一套从混乱到有序的设计框架；对无状态 LLM 来说，这一章提供的是最小可复现的任务重写协议。核心标准只有一个：读完并执行本章后，研究者应当能够明确自己在研究什么、不研究什么、先验证什么、需要什么信息，以及不同时间敏感度的问题该采用哪种设计路径。

模糊问题之所以难，不是因为它“深”，而是因为它通常把多个层次混在一起。一个原始提问里，常常同时混有目标、事实、因果、价值判断、时间范围、主体范围与隐含比较对象。如果不先拆解，后续检索再多，也只是围绕混乱的问题继续堆积材料。因此，研究设计的第一步不是立刻搜，而是先把原问题改写成一个可以操作的研究对象。

可执行的问题必须至少回答五件事：研究对象是谁，研究的现象或命题是什么，时间范围在哪里，空间或情境范围在哪里，判定“回答足够成立”需要满足什么条件。原始问题如果缺少这五项中的任意一项，就需要先补全或暂定。例如，“为什么某行业最近不景气”并不是一个可直接执行的问题，因为行业、时间、指标和比较基线都没有明确。只有把它改写成类似“在过去 12 个月内，某行业的营收、订单、投资或就业指标是否显著下滑；若下滑，主要驱动因素分别是什么”，研究才真正开始有边界。

问题拆解可以按四层进行。第一层是主问题，回答“最终要形成什么判断”。第二层是子问题，回答“这个判断由哪些必要部分组成”。第三层是可检验命题，回答“每个子问题要被什么事实支撑或推翻”。第四层是待取证事项，回答“为了验证命题，下一步要找什么材料”。如果一个问题还停留在抽象讨论层面，就继续往下拆；如果一个问题已经能直接映射到具体信息需求，就说明拆解粒度足够。

一个稳健的拆解结果通常具有“树状而非散点”的结构。主问题位于顶层，子问题之间彼此区分但共同服务于主问题，可检验命题则对应每个子问题的关键判断点。这里要避免两类常见错误。第一类错误是把检索动作当作子问题，例如“去找新闻”“查论文”并不是研究问题，它只是动作。第二类错误是把结论预设成问题，例如“某政策失败的原因是什么”已经偷渡了“失败”这个结论，更合适的写法应是“某政策是否达到其目标；若未达到，主要限制因素是什么”。

在完成问题拆解之后，进入假设驱动（hypothesis-driven）阶段。假设的作用不是提前决定答案，而是为研究提供第一批可被证伪的方向。没有假设，研究容易漫游；只有单一假设，研究又容易变成自我证明。更合理的做法，是同时保留一个主假设和若干替代假设。主假设代表当前最值得先检验的解释，替代假设代表可能与主假设竞争、修正或推翻它的解释路径。

一个可用的研究假设必须满足三个条件。第一，它必须可证伪，也就是存在某类证据一旦出现，就足以削弱或推翻该假设。第二，它必须带出可观察预测，也就是如果假设为真，研究者应当在数据、文本、叙述或时间序列中看到什么现象。第三，它必须与研究边界一致，不能把范围外因素无限纳入。例如，当研究“某产品销量下降是否主要由价格调整引起”时，主假设可以是“价格上涨是销量下降的主要驱动因素”，替代假设则可能包括“渠道收缩更关键”“竞争对手促销影响更大”“季节性波动导致误判”。每个假设都需要对应后续可观察的证据方向，而不是只停留在语言层面的猜测。

为了让假设真正进入执行层，必须给每个假设补上一组“验证信号”。验证信号可以理解为：若假设为真，最可能出现哪些事实；若假设为假，又最可能看到哪些反向迹象。这样做的价值在于，研究者不再只是被动接收材料，而是主动判断某条信息到底增强了哪个假设，还是削弱了哪个假设。对无状态 LLM 尤其重要，因为它需要依靠显式结构而不是会话记忆维持推理方向。

接下来是研究路径设计。研究路径不是来源清单，而是“先看哪类维度，再看哪类维度”的行动顺序设计。一个模糊问题通常可以从五个常用维度展开：时间维度、主体维度、机制维度、约束维度与比较维度。时间维度用于判断变化是否真实发生、何时开始、是否存在阶段性转折。主体维度用于识别谁在行动、谁受影响、谁提供解释、谁承担后果。机制维度用于回答“为什么会这样”，也就是因果链中关键环节如何连接。约束维度用于识别法律、资源、制度、成本、能力或环境限制。比较维度用于防止孤立观察，例如与历史版本、对照地区、同类对象或替代方案比较。

设计研究路径时，不必一次铺满所有维度，但必须明确先后关系。通常更可靠的顺序是：先确认现象是否成立，再追问解释；先确认边界，再扩展机制；先排除定义混乱，再进入因果讨论。原因很直接：如果连现象本身都没有被界定，过早讨论原因只会制造伪解释。因此，研究设计要尽量保持快乐路径（happy path）直线化。先用最少动作确认是否值得深入，再决定是否扩展到更昂贵的研究路径。

围绕研究路径，信息需求也需要被显式分类。第 2 章至少区分五类信息需求。第一类是定义型信息，用于确认概念、指标、口径与边界，防止大家谈论的是同一个词却不是同一个对象。第二类是事实型信息，用于确认事件、数据、时间点、参与方、版本变化等基础事实。第三类是关系型信息，用于识别主体之间、变量之间或事件之间的关联结构。第四类是因果型信息，用于判断某个机制是否真的对结果产生驱动，而不只是伴随出现。第五类是趋势型信息，用于判断变化是短期波动、周期现象还是结构性转折。

这五类信息需求不是平行堆料，而是有依赖关系。定义不清，事实会混乱；事实不稳，关系判断会漂移；关系未澄清，因果推断容易越界；没有趋势视角，偶然波动会被误读成长期规律。因此，当研究受阻时，不要立刻增加更多材料，先检查卡住的是哪一类信息需求。很多所谓“搜索不到”，本质上不是资料不足，而是问题设计没有把信息需求拆干净。

时效性分支必须在研究设计阶段就显式保留，而不是等搜到一半再临时补救。因为同一个问题，只要时间敏感度不同，研究路径就会明显不同。若问题属于高度相关（high relevance）时效场景，例如实时新闻、价格波动、版本更新、安全事件、突发政策变化，研究计划应优先考虑最新时间戳、一手通告、版本记录与连续更新机制，并缩短背景材料的权重。若问题属于中度相关（medium relevance）场景，例如季度表现、年度趋势、阶段性政策执行，则研究计划需要同时保留当前信息与近期历史信息，用来判断当前变化究竟是延续还是拐点。若问题属于低度相关（low relevance）场景，例如行业机制、制度演化、组织惯例，则可以提高背景材料、长期样本与历史对照的比重。若问题属于时间无关（time-invariant）场景，例如基础定义、稳定理论、数学关系，则研究设计应优先保证概念准确性与经典权威性，而不是过度追逐最新来源。

时效性分支的核心不是“新的一定更好”，而是“时间是否是结论成立条件的一部分”。如果时间会直接改变结论，就必须把时间放进主问题与假设中；如果时间主要影响材料丰富度但不改变结论逻辑，可以把时间降为背景条件。对无状态 LLM 来说，最安全的做法是每次在开始研究前明确写出当前问题属于哪一类时效分支，并说明这一判断如何影响后续路径。

将以上步骤收束起来，一个完整的第 2 章输出应该至少形成如下研究设计骨架：原始问题重述；研究边界；主问题与子问题；主假设与替代假设；每个假设的验证信号；优先研究路径；所需信息类型；时效性分类与对应处理策略。只要这些元素齐备，研究者就已经拥有了一份可执行、可检验、可迭代修正的计划。此时即使还没有开始取证，也已经从“模糊提问”跨入“研究状态”。

若通过 research CLI 执行，本章产物应先被压缩成可落盘的最小输入（minimal input）：`title`、`question`、当前边界、主假设与下一步研究路径。正确顺序是先完成第 2 章，再调用 `init`、`node_add` 或 `research_search`；不要先跑命令、后补研究设计，否则 CLI 只会把混乱状态稳定地写进系统。

可以用一个最小自检来判断本章是否执行到位。如果把当前研究计划交给另一个没有上下文的人类或无状态 LLM，对方是否能回答以下问题：现在到底要验证什么命题；先验证哪一步；什么证据会支持或削弱当前假设；缺的是什么类型的信息；这个问题是否受时间强烈影响。如果这些问题仍答不出来，说明研究设计尚未完成，不应跳到后续取证阶段。

### 第 3 章 取证、验证与证据链构造

- 章节职责: 说明如何找证据、评估来源、处理冲突并构造证据链。
- 必须覆盖: 广搜与深搜、来源分级、交叉验证、事实核查流程、知识缺口与知识盲区。
- 禁止越界: 不在本章展开停止条件或最终输出模板。

<!-- chapter:03:start -->
第 3 章解决的是执行层最关键的问题：证据从哪里来，哪些能采信，冲突信息怎样处理，最后又怎样把零散材料组织成一条可复核的证据链（evidence chain）。如果第 2 章负责把问题设计清楚，那么第 3 章负责把研究真正落到材料世界中。对人类读者来说，本章提供的是取证与核验的操作逻辑；对无状态 LLM 来说，本章提供的是一套避免“看见材料就急着下判断”的约束协议。核心要求只有三条：先找全版图，再深挖关键点；先判来源与独立性，再判结论；先组织证据关系，再书写判断。

研究中的“找证据”，从来不是把搜到的材料越堆越多，而是持续回答三个判断问题。第一，这条材料回答的是哪个命题。第二，这条材料属于原始证据、二手转述，还是只是线索。第三，这条材料进入论证后，是增强、削弱，还是仅仅改变了问题边界。不能回答这三个问题的材料，即使内容看起来丰富，也只能算噪声（noise），不能直接进入结论层。

### 3.1 从广搜到深搜：先建立信息地图，再打通关键断层

广搜（broad search）的职责不是定案，而是建图。刚进入一个问题时，研究者通常只知道提问表面，不知道真正的争议点、关键主体、主要时间节点、核心概念口径和潜在反向解释。此时如果直接扎进某个单点材料，很容易被最先看见的叙事带偏。广搜的目标，是尽快把问题周围的“信息地形”看清楚，至少识别出以下五类元素：

- 这个问题涉及哪些主体，谁在陈述，谁在行动，谁受影响。
- 这个问题的关键时间点是什么，是否存在版本变化、政策变化、市场变化或事件拐点。
- 主张之间的主要分歧在哪里，是事实冲突、定义冲突，还是解释冲突。
- 哪些来源反复出现，哪些来源只是互相转引。
- 当前明显缺什么，是缺基础定义、缺关键数据、缺原始记录，还是缺反证路径。

因此，广搜的正确产出不是“我已经知道答案”，而是一张信息地图（information map）。这张地图至少要标出四种区域：已经有多源覆盖的区域、只有单一来源支撑的区域、不同说法相互冲突的区域，以及尚未被触及的空白区域。广搜做完后，研究者应该能够说清楚“下一步该深挖哪一个点”，而不是继续盲目扩大搜索面。

深搜（deep search）的职责，是打通广搜暴露出来的路径断层（path discontinuity）。路径断层指的是：当前论证已经走到某一步，但中间缺少关键证据，导致事实无法稳定地连接到判断。深搜不是“搜得更久”，而是“搜得更窄、更深、更接近原始材料”。一次合格的深搜，只围绕一个高价值目标展开，例如一个关键定义、一个时间节点、一份原始文档、一条核心数据、一段视频原片、一个声明的最早版本，或者一条争议说法的首发来源。

广搜与深搜之间应当允许反复切换。广搜发现了新主体或新争议点，就回到建图状态；深搜拿到关键原件后，又可能暴露新的空白与矛盾，需要重新扩图。这里的判断标准不是“已经搜了多少轮”，而是“当前动作的信息增益（information gain）是否仍然明显”。如果继续深挖某一路径只会得到更多重复转述，而不能增加独立证据，就应退出当前深搜，回到广搜或转向其他路径。

关键词设计也应服务于这个广搜到深搜的转换。起步时优先使用单一焦点词组，用来识别基本版图；随后围绕同义词、上下位概念、制度口径、时间约束、主体约束和机制约束逐步收窄。一个稳健的节奏通常是：先用宽词发现版图，再用窄词定位节点，最后用精确限定词锁定原文。关键词一旦堆得过多，往往意味着研究者还没分清自己到底在验证哪个命题。

还要明确一个硬规则：当某条信息将被用作关键判断依据时，不能停留在摘要依赖（summary dependence）层。凡是涉及定义边界、数据口径、制度文本、法律义务、争议引语、版本差异、图像来源或因果解释的关键材料，都必须尽可能回到原文、原始记录或原始载体。摘要可以帮你定位材料，不能替你完成采信。

### 3.2 来源分级：判断可信度，不等于迷信权威

来源分级（source grading）的作用，不是给来源贴永久标签，而是帮助研究者在有限时间内决定“先看谁、信到什么程度、还需要谁来交叉验证”。来源可信度始终是动态判断（dynamic assessment），因为同一来源在不同主题、不同时间、不同利益结构下，表现可能完全不同。一个可靠的分级判断，至少同时看四个维度：身份与职责、历史记录、利益动机、可验证性。

- 身份与职责：来源是否对其陈述对象有直接记录职责、管理职责、调查职责或专业解释职责。
- 历史记录：该来源过去是否持续准确，是否有明显撤稿、误报、夸张叙事或选择性呈现历史。
- 利益动机：该来源是否直接从某种叙事中获利，是否存在立场、市场、政治或声誉驱动。
- 可验证性：该来源是否提供可复核的原始材料、数据口径、时间戳、方法说明，是否允许外部独立检验。

为了快速进入执行状态，本技能采用五级来源分级表，但要记住，这个等级只表示“默认起点”，不表示“可以跳过验证”。

| trust_level | 默认可信度 | 典型来源特征 | 默认处理方式 |
| --- | --- | --- | --- |
| 5 | 最高 | 国际权威机构、国家级正式记录机构、专业学术或医疗权威组织、原始法规与正式统计发布者 | 优先纳入，但仍检查口径、发布日期和适用范围 |
| 4 | 较高 | 官方或半官方机构、主流通讯社、专业出版社、正式行业机构 | 可作为强支撑来源，需补充至少一个独立来源核对 |
| 3 | 中等 | 主流数字媒体、普通报刊、门户站点、一般行业媒体 | 适合作为线索与背景，不能单独支撑高置信结论 |
| 2 | 较低 | 专门领域媒体、小型机构、自带强立场的评论平台、研究简报 | 可用来发现议题与争议点，必须回溯更高质量来源 |
| 1 | 最低 | 匿名帖文、二次转载、无出处截图、断章视频、口耳转述 | 仅当作待核查线索，不得直接进入事实结论 |

需要特别防止两个常见误判。第一，把“高等级来源”误当成“不会错”。即使是高等级来源，也可能因为统计口径、发布时间、翻译处理、局部观察或初始通报限制而不完整。第二，把“低等级来源”一概扔掉。低等级来源虽然不能直接采信，但常常能暴露未被注意的线索、反例、版本差异或利益冲突。研究的正确做法不是迷信权威，而是利用来源分级决定取证顺序和验证强度。

真正进入证据链的，不是“某来源说了什么”这一句话，而是“某来源在什么位置、基于什么材料、在什么时间、以什么口径说了什么”。一旦这些元信息（metadata）缺失，材料就很难被稳定地放进论证结构。

### 3.3 交叉验证与证据链构造：从材料堆积变成可复核论证

交叉验证（cross-validation）的底线，是同一主张至少获得两个独立来源（independent sources）的支持，才能进入“初步可信”状态。但“两个来源”只是下限，不是充分条件。真正的关键在独立性。如果两个报道都转引同一份新闻稿、同一段视频剪辑、同一名未披露身份的爆料者，它们在逻辑上仍然只是一条证据链路，不能算两份独立证据。

判断独立性时，至少检查三件事。第一，信息起点是否相同。第二，验证过程是否相互独立。第三，叙述结果是否只是同一文本的改写。只要上游错误源相同，下游材料再多，也只是在重复同一个错误。

在独立性满足底线之后，再做一致性检验（consistency check）。一致性不只是看结论是否相似，而是要沿着事实的关键维度逐项核对：

- 时间是否一致，是否说的是同一时间段、同一版本或同一事件阶段。
- 地点是否一致，是否把不同区域、不同平台、不同制度环境混为一谈。
- 主体是否一致，是否把组织、部门、子公司、代理人或个体混写成同一个行动者。
- 指标口径是否一致，是否把不同统计口径或不同定义的数字强行并列。
- 机制解释是否一致，是否只是现象一致而因果链条完全不同。

还要把材料明确分层。事实（fact）是可直接观察、记录或核验的内容；推断（inference）是基于事实做出的中间判断；观点（opinion）是价值立场、解释框架或主张偏好。证据链最容易失真的地方，就是把观点写成事实，把推断伪装成观察。因此，每条材料进入证据链之前，都应被标注为支持性证据、反驳性证据、背景性证据或待核线索，并写清它服务的是哪个子命题。

一条可复核的证据链，至少应包含五个字段：证据内容、来源与链接、时间戳或版本、可信度等级、在论证中的作用位置。只要缺少其中两项以上，这条材料就不适合承担关键支撑职责。研究者真正要构造的，不是“资料库”，而是“命题到证据的映射关系”。也就是说，每个子命题后面要明确挂接哪些证据、这些证据彼此是什么关系、有没有反证、当前链条哪一段仍然悬空。

当出现冲突信息时，不要急着选边站，而要先分类冲突类型。常见冲突有四种。第一，事实冲突：双方对是否发生、数量多少、时间何时发生给出不同说法。第二，定义冲突：双方用的是不同口径，却误以为在讨论同一件事。第三，范围冲突：局部样本被误当成整体，旧版本被误当成当前版本。第四，解释冲突：事实相近，但因果说明不同。只有先识别冲突类型，才能决定下一步是补原始记录、补背景定义，还是补替代解释。

### 3.4 事实核查专项流程：专门处理争议主张

当研究对象是一条争议主张（contested claim），例如“某人说过某句话”“某机构实施了某政策”“某事件造成了某结果”，普通取证流程通常不够，因为这类主张的核心难点不在材料多少，而在版本漂移、转述失真和证据伪装。此时应进入事实核查（fact-checking）专项流程。

第一步，先把主张改写成最小可核单元。不要核查一个模糊的大句子，而要拆成若干可以分别被证实或证伪的原子命题。例如一条流传很广的说法，可能同时包含时间、地点、主体、动作、结果五个元素，必须逐一核，不要整体吞下。

第二步，追首发与版本演变。先问这条说法最早从哪里出现，后续传播中哪些词发生过变化，哪些截图、转述、剪辑或二次概括扩大了原意。很多争议并不是源头为假，而是传播链在每一轮转述中逐步变形。只要版本链没理清，就很难知道自己到底在核查哪一个命题。

第三步，优先寻找一手材料（primary evidence）。一手材料包括原始文档、正式公告、原始数据、完整视频、现场图像、庭审记录、法规文本、论文原文、数据库记录、原始访谈录音等。只有在一手材料不可得时，才提高二手高质量来源的权重；即便如此，也要明确说明自己仍处于“未见原件”的受限状态。

第四步，建立时间轴（timeline）。把所有关键证据按时间排序，标出事件发生、材料发布、版本变更、回应出现和外部报道进入的顺序。时间轴的价值在于，它能立刻暴露很多伪冲突与伪因果。一个在结果之后才出现的解释，不应被写成原因；一份在版本更新之前发布的旧文件，也不能直接代表当前状态。

第五步，做反证搜索（counter-evidence search）。事实核查不是只寻找支持某个结论的材料，而是主动寻找最可能推翻当前判断的证据。如果你暂时相信某主张为真，就去搜什么材料最可能证明它为假；如果你暂时相信它为假，就去搜什么材料最可能证明它被低估或误解。没有经过反证搜索的核查，很容易沦为立场确认。

第六步，写出核查边界。事实核查的合格结果，不只是“真”或“假”，还包括“哪一部分已证实，哪一部分未证实，哪一部分因证据不足仍保持开放”。只要核查过程存在关键原件缺失、版本链断裂、数据口径不明或主体身份未确认，就必须把这些限制连同判断一起交付，而不是藏起来制造确定感。

### 3.5 知识缺口与知识盲区：知道哪里没补齐，也知道哪里根本还没看到

研究在执行中常常会卡住，但卡住的原因并不相同。知识缺口（knowledge gap）指的是：研究者已经知道某一段论证需要什么证据，只是当前还没有拿到。例如，已经知道要判断一项政策是否生效，需要月度数据和实施细则，但这两项材料尚未取得。知识盲区（knowledge blind spot）则更危险，它表示研究者还没意识到自己遗漏了哪个方向，或者根本没有把某个可能改变结论的变量纳入版图。

两者的处理方式必须分开。面对知识缺口，正确动作是定向补齐：补数据、补定义、补原件、补独立来源、补反证。面对知识盲区，正确动作不是硬搜当前路径，而是承认当前地图可能不完整，然后有意识地换维度重看问题，例如改从其他主体、其他时间段、其他制度层级、其他学科解释或其他利益相关方切入。

可以用一个简单判据区分两者。如果你能明确说出“我还缺哪一块证据才能把这段论证接上”，那通常是缺口；如果你只能感觉“结论总有点不稳，但说不清还漏了什么”，那往往已经碰到盲区。缺口会阻断证据链，盲区会扭曲证据链。前者是链条不完整，后者是地图不完整。

广搜与深搜在这里也承担不同职责。广搜主要用于发现盲点，因为它帮助你看到版图之外可能还存在什么方向；深搜主要用于修补断层，因为它帮助你把已知路径中的关键环节挖实。当一个研究在同一路径上继续投入却几乎不再增加有效证据时，要警惕自己不是“还差一点”，而是已经被盲区困住。

本章的目标，是建立一套稳定的取证纪律。先用广搜看清版图，再用深搜补强关键链路；先对来源分级，再决定采信力度；先做独立性与一致性检查，再进入结论；面对争议主张时，优先核查首发、版本与原件；在研究受阻时，区分自己面对的是知识缺口还是知识盲区。只要这些动作被持续执行，证据链就会逐步长成一套可解释（explainable）、可复核（reviewable）、可迁移（transferable）的判断结构。

### 第 4 章 反思循环、停止条件与降级策略

- 章节职责: 管理研究过程，定义何时继续、何时停止、何时降级。
- 必须覆盖: 三段式反思、停止条件、残余不确定管理、工具失败降级。
- 禁止越界: 不重复第 2 章设计步骤与第 5 章输出模板。

<!-- chapter:04:start -->
第 4 章处理的不是“还要不要继续搜”这种直觉问题，而是研究过程本身的控制问题。前 3 章解决了什么值得研究、怎样设计问题、怎样取证与验证，但一个研究真正失控，往往不是因为没有方法，而是因为研究者不知道什么时候该继续、什么时候该停、什么时候该承认当前条件已经不足以支持更深入行动。对人类读者来说，本章给出的是过程控制协议；对无状态 LLM 来说，本章给出的是每轮动作后必须执行的状态更新机制。核心要求很简单：每一轮动作结束后，都要显式判断当前状态属于继续（continue）、停止（stop）还是降级（degrade），而不是靠感觉拖延或草率收束。

### 4.1 三段式反思：每轮动作之后都要更新研究状态

深度研究要求每一次关键动作之后都立即反思。这里的动作可以是一次搜索、一次原文抓取、一次交叉比对、一次版本核验，也可以是一次结构化整理。只要动作产生了新信息，研究状态就可能变化。没有这一层反思，研究就会在材料堆积中逐渐失去方向。

每轮反思都必须包含三个部分，而且顺序不能颠倒。

第一部分，记录新证据或新要点。重点不是复述刚刚看见了什么，而是说明这次动作究竟改变了什么。一个合格的记录至少要回答：这条信息支持了哪个命题，削弱了哪个命题，还是只是改变了研究边界。若结果只是重复已有材料，也要明确写出“没有新增独立证据”，因为重复本身就是关于信息增益下降的重要信号。

第二部分，更新假设可信度并标出仍存的不确定。这里不是要求做精确概率计算，而是要求研究者明确表达：当前主要判断比上一轮更强、更弱，还是基本不变；导致这一变化的关键证据是什么；仍未闭合的链路在哪里。对无状态 LLM 最稳妥的做法，是把判断变化写成三类之一：增强、削弱、未变。随后再补一句原因和剩余不确定来源。剩余不确定可以来自证据缺口、定义歧义、时间版本未对齐、来源独立性不足、反证尚未检验，或潜在知识盲区尚未排查。

第三部分，定义下一轮最小高信息增益动作。这里的关键词是“最小”和“高信息增益”。下一步不应是模糊的“继续研究”，而应是一个单一、可执行、可验证的动作，例如：补抓某份原文、核对某个时间节点、为一条冲突说法寻找独立来源、验证某个替代解释是否成立，或在证据已足够时停止主动搜索并进入结论收束。只要下一步动作不能被明确说成一句可执行指令，就说明反思还没有完成。

可以把三段式反思压缩为一个固定协议。先写“本轮新增了什么”，再写“它如何改变当前判断，还有什么没解决”，最后写“下一步唯一优先动作是什么”。这三个部分共同承担状态更新职责：第一部分负责吸收新信息，第二部分负责更新认知位置，第三部分负责限制行动范围。没有第三部分，研究会扩散；没有第二部分，研究会漂移；没有第一部分，研究会自我重复。

如果配合 research CLI，这一层最适合映射到显式状态更新（explicit state update）：用 `node_update`、`node_resolve`、`evidence_verify`、`run` 或 `graph_snapshot` 记录本轮结果，但要记住，CLI 只负责承载状态，不负责替你决定 `continue`、`stop` 或 `degrade`。状态判断仍然必须来自本章的反思协议，而不是来自某个命令是否执行成功。

研究过程中的状态判断也应直接从三段式反思得出。如果当前仍有关键问题未闭合，且下一步存在清晰、可行、预期信息增益较高的动作，状态应标为继续。如果关键链路已经闭合，或新增动作大概率只会带来重复材料，状态应转为停止。如果真正卡住的是工具、权限、成本或外部条件，而不是逻辑上仍有必要继续，则状态应转为降级。换言之，三段式反思不是记录工具，而是驱动状态机（state machine）。

### 4.2 停止条件：研究结束不是因为完美，而是因为边际增益已经改变

深度研究的停止条件，应理解为：在当前目标、当前资源与当前边界下，继续主动研究已不再显著提高结论质量。一个研究如果把“完全没有不确定”当作唯一停止标准，通常会无限拖延；如果把“已经看了不少资料”当作停止标准，则通常会过早收束。正确的停止判断，需要同时看证据充分性、剩余不确定的重要性，以及继续行动的成本。

第一类停止条件，是关键命题已经获得多源一致支持，且主要要素已经齐备。这里的“齐备”至少包括：核心事实已被独立来源支撑，关键定义或口径已对齐，主要替代解释已经被检验过，结论的适用范围已经说清楚。此时即使还存在一些局部细节没有补齐，也不必无限延长研究，因为核心判断已经具备可交付性。

第二类停止条件，是继续研究的边际增益明显下降。常见信号包括：新搜索结果大量重复已有来源；新增材料只是在改写旧信息；继续投入只能补充修辞而不能改变判断；当前尚未解决的问题需要付出远高于其价值的成本才能推进。只要连续几轮动作都无法带来新的独立证据或新的判别力，就应认真考虑关闭主动搜索，而不是机械增加轮次。

第三类停止条件，是研究推进已经触及外部边界。例如关键证据只存在于线下档案、付费数据库、受限系统、私人访问权限、专门实验、法律程序或实地调查中，而当前执行者并不具备这些条件。这种情况下，继续在线上或现有工具中反复打转，并不会让结论更可靠，只会制造“似乎还在努力”的假象。正确做法是停止当前研究循环，清楚交付已经确认的部分、尚未确认的部分，以及下一步需要什么外部条件。

第四类停止条件，是研究目标本身已经达到。深度研究不是总要把问题解释到最深层机制才算结束，它首先服务于当前任务。如果任务目标只是支持一次决策、完成一次核查、形成一份风险判断或给出下一步行动建议，那么当现有证据已经足以支撑该任务目标时，研究就可以停止。把研究做得比任务需要更深，不一定更好，反而可能浪费时间并引入额外噪声。

因此，一个稳健的停止判断至少要同时满足两个问题的回答。第一，当前证据是否已经足以支撑本轮任务目标。第二，继续研究是否仍有明显、现实、可负担的信息增益。如果第一问为是，且第二问为否，应停止。如果第一问为否，但第二问也为否，说明研究不是应当继续，而是应当转入降级交付，因为当前条件已经不足以支持再前进。

### 4.3 残余不确定管理：区分可接受不确定与致命缺口

停止研究并不等于没有不确定。真正需要管理的，不是不确定本身，而是不确定是否足以改变结论。很多研究错误并不是因为存在未知，而是因为没有区分“可以带着走的未知”和“会推翻结论的未知”。因此，在每次准备停止时，都必须显式处理残余不确定（residual uncertainty）。

可接受不确定，指的是即使未来补到更多材料，也更可能只会细化结论、修正表述边界或调整置信度，而不会根本逆转当前核心判断。例如某个背景变量尚不充分、某个次级时间点仍缺精确记录、某些非关键细节未能完全对齐，但这些不足不会动摇主命题成立与否。这类不确定应被保留、披露，但不必阻止交付。

致命缺口则不同。它指的是某个尚未解决的问题一旦方向相反，就足以改变主结论、推翻当前解释、改变适用范围，或使证据链核心环节失效。常见的致命缺口包括：关键来源并不独立、核心定义其实未对齐、主要数据口径错误、反证路径从未检验、时间版本严重错位、关键因果链条仅有相关性没有支撑。如果这些问题仍未处理，就不应把当前结果包装成高置信结论。

管理残余不确定时，最重要的是把它与结论绑定，而不是单独放在尾部做礼貌性声明。更安全的表达方式是：当前结论在哪些前提下成立；哪些未证实因素仍可能改变它；若这些因素未来被证实为相反方向，最可能推翻的是哪一部分。这种写法的价值在于，它让读者或无状态 LLM 能立刻看懂当前结论的承重结构，而不是只看到一个抽象的“仍有不确定”。

残余不确定的处理结果，应直接影响状态判断。如果剩余的只是可接受不确定，可以停止并交付条件化结论。如果剩余的是致命缺口，但当前仍存在高信息增益且可执行的验证动作，应继续。如果剩余的是致命缺口，但已无现实可行的验证动作，应降级交付，明确说明当前只能给出低置信或条件性判断，而不能伪装成稳定结论。

### 4.4 工具失败与条件受限时的降级策略

降级（degrade）不是失败后的情绪性放弃，而是在工具、权限、时间或外部条件受限时，主动把研究切换到一个仍然诚实、仍然可用的输出级别。它的目标不是继续伪装完整研究，而是在不编造、不跳步的前提下，最大化当前已验证信息的使用价值。

工具失败时，先区分是暂时故障，还是结构性不可得。若只是请求超时、单页抓取失败、关键词误设、页面结构异常或短暂网络波动，可以在确认输入无误后有限重试。但重试必须有上限。一个可执行的默认规则是：同一关键动作最多重试两次；若两次后仍失败，就停止在该路径上继续消耗，并将状态转为降级评估。反复重试而不改变策略，不属于研究努力，只属于噪声生产。

一旦进入降级状态，研究者应立刻做四件事。第一，冻结当前已经核实的证据，不再把未成功获取的内容当作已知。第二，下调结论置信度，因为证据链有一部分未被补实。第三，明确写出失败发生在什么环节，例如原文不可抓取、数据库不可访问、权限不足、时效限制过强、线下验证缺失。第四，给出下一步最现实的补救路径，例如改由人工访问、换用更高权限工具、等待公开版本、寻求领域专家、查询线下档案，或把问题拆成更小的可验证部分。

降级后的产出，仍然必须满足真实性约束。它可以交付当前最佳可核实结论，可以交付条件化判断，可以交付“哪些部分已确认、哪些部分未确认、为什么未确认”，但不能把猜测补成事实，不能用语气强度弥补证据缺失，也不能因为工具失败就把未验证的主张直接收进结论。对无状态 LLM 尤其要强调这一点：工具失败只改变可确认范围，不授权模型用想象填补缺口。

本章建立的是一个简单但严格的过程控制框架。每轮动作后，先用三段式反思更新研究状态；随后根据证据充分性、边际增益与外部边界判断是继续、停止还是降级；准备停止时，先检查剩余的是可接受不确定还是致命缺口；一旦工具或条件失效，就在有限重试后切换到诚实的降级交付。只要这套机制持续执行，研究过程就不会被材料数量绑架，也不会因为条件受限而滑向伪确定答案。

### 第 5 章 输出模板、复核清单与附录工具箱

- 章节职责: 将前四章方法收束成可直接调用的输出模板与复核工具。
- 必须覆盖: 结构化回答模板、纪实式事实核查写法、自我复核清单、快速参考卡。
- 禁止越界: 不回头重写前四章理论主体，只做收束与落地。

<!-- chapter:05:start -->
第 5 章不再解释“为什么要这样研究”，而是把前四章已经建立的方法压缩成可以直接调用的工作模板（working templates）与复核工具。它服务的对象有两个：一类是已经理解前四章框架、现在要立刻开始一次研究任务的人类研究者；另一类是没有历史记忆、必须依赖显式结构执行任务的无状态 LLM。无论是哪一类使用者，第 5 章都只回答一个问题：当研究真的开始时，应该怎样稳定地产出可交付结果，而不是重新回到概念讨论。

### 5.1 结构化回答模板：把研究结果写成可检查的交付物

一份合格的深度研究输出，不只要给结论，还要交付结论边界、支撑证据和未解决部分。最稳妥的默认模板如下，字段本身不建议随意删减。

```md
## 问题重述

- 研究问题: <用一句话重述原问题>
- 研究边界: <主体 / 时间 / 地点 / 版本 / 不纳入范围的部分>
- 任务目标: <本次研究要支持什么判断或行动>

## 研究路径

1. <第一轮做了什么，为什么先做这一步>
2. <第二轮新增了什么证据，策略如何调整>
3. <第三轮如何处理冲突、缺口或盲区>

## 证据要点

### 来源组 A: <原始记录 / 官方资料 / 一手材料>
- 证据 1: <事实内容>
  - 来源类型: <一手 / 二手>
  - 默认可信度: <5 / 4 / 3 / 2 / 1>
  - 在论证中的作用: <支撑 / 反驳 / 背景 / 待核>

### 来源组 B: <独立媒体 / 专业分析 / 对立观点>
- 证据 2: <事实内容>
  - 来源类型: <一手 / 二手>
  - 默认可信度: <5 / 4 / 3 / 2 / 1>
  - 在论证中的作用: <支撑 / 反驳 / 背景 / 待核>

## 推理链

1. 已确认事实: <不带推断色彩的事实>
2. 中间判断: <这些事实共同支持了什么判断>
3. 结论: <当前最稳妥的结论>
4. 关键前提: <结论成立依赖哪些前提>

## 结论确定度

- 确定度: <High / Medium / Low>
- 判断依据: <为什么只能到这个等级>

## 残余不确定

- 尚未闭合的缺口: <仍未补齐的关键点>
- 潜在盲区: <可能还没充分展开的方向>
- 若继续研究，最优先下一步: <一个最小高信息增益动作>
```

这个模板的关键，不在于“填满所有格子”，而在于强制区分四件事：问题边界、证据内容、推理过程、结论强度。结构化模板的价值，就是把这四层明确拆开。

若使用 research CLI，可以把这个模板理解成命令链的输出映射（output mapping）：`init` 对应问题重述与任务目标的起点，`node_add` 和 `graph_link` 对应研究路径与推理链，`evidence_add` / `evidence_verify` / `evidence_link` 对应证据要点，`run` 与 `graph_snapshot` 对应阶段状态，`export` 对应最终交付。一个可用的最小顺序通常是 `init -> node_add -> evidence_add -> evidence_verify -> evidence_link -> graph_snapshot -> export`。

如果研究对象是开放问题而非争议主张，模板可以稍作压缩，但不应删除“残余不确定”与“结论确定度”两项。没有这两项，输出就会滑向伪确定答案。反过来，如果研究对象是高争议、高利害或高时效问题，则应加强“研究路径”和“证据要点”两部分，明确写出版本差异、来源独立性与时间范围。

### 5.2 纪实式事实核查写法：克制叙述，不把判断伪装成事实

当输出对象是一条争议主张时，列表式模板未必最适合对外阅读。此时更稳妥的写法，是采用纪实式（documentary-style）事实核查结构：按调查顺序交代主张、溯源、核验过程、关键证据、冲突点和最后判断，让读者顺着路径看见“为什么这个结论成立”。

纪实式写法有四个要求。

第一，开头先交代被核查的最小命题，而不是先给价值判断。不要一上来写“这是假消息”或“这一说法成立”，而应先写“本次核查的对象是什么，核心争议在哪个事实上”。这样可以避免把结论偷渡进问题。

第二，正文按证据进入的顺序推进，而不是按情绪强弱推进。先写最早出现的版本、首发源头或原始记录，再写传播过程中的变化，接着写如何找到了关键原件、关键数据或独立来源，最后再解释哪些证据支持、哪些证据削弱当前主张。这样写，读者能直接看到调查路径，而不是被动接受终点判断。

第三，语气必须克制（restrained）。克制不等于冷漠，而是避免夸张、嘲讽、先验立场和语义偷换。纪实式事实核查最常见的写作失败，是用很强的措辞掩盖证据并不充分。只要证据链里仍有未闭合部分，语言就应当保留条件性，而不是用语气代替验证。

第四，结尾要明确区分“已经证实”“尚未证实”“无法证实”。事实核查不是只有真与假两档，很多时候更准确的表达是：某一部分已被原始材料证实，某一部分因版本漂移被削弱，某一部分目前缺乏足够证据，因此不能扩大解释。把这种边界交代清楚，比给出一个过于整齐的标签更重要。

可以直接采用下面这个纪实式骨架。

```md
这次核查聚焦于 <争议主张>。要判断这条说法是否成立，首先需要确认 <最小可核命题>，因为传播过程中它已经出现了 <版本变化 / 范围扩张 / 语义漂移>。

沿着传播链回溯后，可以先确认 <首发来源 / 最早版本 / 原始记录>。这一步说明 <哪部分说法确实出现过，哪部分是后续传播中被加上的>。随后再比对 <原始材料 A> 与 <独立来源 B>，可以看到二者在 <时间 / 主体 / 口径 / 数据> 上是 <一致 / 部分冲突 / 明显冲突>。

决定当前判断的关键证据是 <关键证据>。它表明 <事实层结论>。但同时仍有 <未解决限制>，因此更稳妥的写法不是“完全证实”或“完全证伪”，而是：<条件化判断>。

基于现有材料，可以确认的是 <已确认部分>；不能确认的是 <未确认部分>；如果还要进一步提高结论确定度，下一步最值得补的是 <下一步验证动作>。
```

纪实式写法适合用于事实核查、争议澄清、事件回溯、版本变迁分析等任务；不适合用于纯定义查询、简单资料汇总或低复杂度说明。写法的选择，应服务于问题结构。

### 5.3 自我复核清单：在交付前检查你有没有跳步

研究结果真正失真，往往发生在写完之后。因为一旦开始输出，人就会自然倾向于把链条补齐、把语气抬高、把未确认部分写得像已经成立。自我复核清单（self-review checklist）的作用，就是在交付前强制做最后一次反向检查。

交付前至少逐项检查以下内容。

- 问题边界是否写清楚了。读者能否一眼看懂本次研究研究了什么、没有研究什么。
- 关键结论是否都能追到证据。任一重要判断都不应只靠语气、直觉或单一转述成立。
- 支撑性证据与反驳性证据是否都被看见。若只保留支持当前观点的材料，结论可信度应自动下调。
- 来源是否足够独立。若两个来源都回到同一上游材料，就不能当作双重验证。
- 时间、主体、地点和口径是否已经对齐。若这些元信息未对齐，很多“矛盾”只是伪冲突。
- 事实、推断、观点是否分层表达。尤其要防止把解释性语言写成事实描述。
- 结论确定度是否和证据强度相匹配。证据只到中等，就不要写成高确定度。
- 残余不确定是否已经披露。没有披露限制条件的研究，通常不是真正稳健的研究。
- 下一步建议是否具体。若建议只是“继续研究”，说明当前输出仍不够结构化。

如果其中任一项无法明确回答，最安全的处理不是硬交付，而是回到前面的模板，补出那一段缺失结构。深度研究要求的是可复核，不是看起来完整。

为了提高自检效率，可以在交付前再问自己三个压缩问题。第一，我的核心结论若要被推翻，最可能被哪条反证推翻。第二，如果把这份输出交给另一个无状态 LLM，它是否能凭这份文本复现我的判断路径。第三，若读者只读“结论确定度”和“残余不确定”，是否也能理解为什么当前不能说得更满。只要这三个问题答不稳，输出就还没有完全收束。

### 5.4 从单次研究到能力沉淀：建立个人工作模板与案例库

第 5 章的最后一步，是提醒研究者把高频动作沉淀下来。深度研究的难度，并不只来自单次任务，也来自每次都从零开始。成熟研究者与新手的差异，往往不在智力，而在是否已有稳定的模板、检查点和失败经验库。

最值得沉淀的内容通常只有三类。第一类是问题拆解模板，也就是你常处理的问题类型有哪些，它们通常需要哪些子命题与信息需求。第二类是证据判断模板，包括哪些来源在你的领域里通常更可靠、哪些口径最容易误读、哪些反证路径必须常规检查。第三类是案例库（case library），记录典型成功案例、典型误判案例、常见版本漂移模式、常见伪冲突类型与常见致命缺口。

沉淀的目的，是缩短从“看见问题”到“进入研究状态”的时间。对个人研究者来说，这意味着把高频 checklist 固化下来；对无状态 LLM 来说，这意味着优先调用一套显式的最小模板。

因此，第 5 章可以被视为整份技能文档的执行面。前四章负责建立边界、设计、取证和过程控制，第 5 章则把这些能力压缩成可复制（repeatable）、可调用（callable）、可复核（reviewable）的研究工具箱。真正的落地，是在每次研究里都能稳定调用这些模板与清单。

## 附录 A 术语表

- 用途: 统一关键术语定义，避免跨章漂移。

<!-- appendix:a:start -->
| 术语 | 定义 |
| --- | --- |
| 深度研究 | 面向复杂、模糊、争议或高利害问题的循证研究活动，目标是构造可复核的判断，而不是快速给出单一答案。 |
| 普通搜索 | 以快速获取事实、定义、入口信息为主的线性检索活动，通常不承担多轮验证与置信度管理。 |
| 证据链 | 支撑某个结论的一组证据及其相互关系，强调来源、时间、作用位置与可追溯性。 |
| 推理链 | 从已确认事实到中间判断再到结论的逻辑路径，用来解释“为什么会得到这个判断”。 |
| 研究边界 | 本次研究明确纳入和排除的主体、时间、地点、版本、问题范围与任务目标。 |
| 主问题 | 本轮研究最终要回答的核心问题。 |
| 子问题 | 为回答主问题而拆解出的必要组成部分。 |
| 假设 | 当前最值得先检验的解释路径或判断方向，必须允许被证据削弱或推翻。 |
| 替代假设 | 与当前主假设竞争、修正或推翻的其他解释路径。 |
| 可证伪 | 一个判断存在潜在反证入口，理论上可以被证据推翻，而不是永远自我成立。 |
| 广搜 | 用于建立信息地图、发现主体、时间点、争议点和潜在盲区的宽范围搜索。 |
| 深搜 | 围绕单一高价值目标展开的窄而深的检索，用于打通路径断层、补关键原件或核对关键口径。 |
| 信息地图 | 对当前问题涉及的主体、时间、冲突点、空白区和已覆盖区的结构化认识。 |
| 路径断层 | 论证走到某一步时缺少关键证据，导致事实无法稳定连接到判断的断点。 |
| 来源分级 | 按默认可信起点对来源进行分层，用于决定取证顺序和验证强度，而不是决定最终真伪。 |
| 独立来源 | 不是从同一上游材料、同一爆料链或同一原始文本改写出来的来源。 |
| 一致性检验 | 对时间、地点、主体、口径、版本和机制解释逐项核对，防止伪冲突和伪一致。 |
| 事实核查 | 围绕争议主张进行溯源、核验原件、建立时间轴和搜索反证的专项流程。 |
| 知识缺口 | 已知论证链中明确缺失的证据、定义、数据或原件。 |
| 知识盲区 | 尚未进入当前问题版图、但可能显著改变结论的未知方向。 |
| 三段式反思 | 每轮动作后都要完成的状态更新协议，包括新增证据、判断变化与下一步最小动作。 |
| 停止条件 | 判断当前研究应结束的标准，通常基于证据充分性、边际增益和外部边界。 |
| 残余不确定 | 研究结束时仍未完全消除、但必须显式披露的未知部分。 |
| 致命缺口 | 一旦方向相反就足以推翻主结论或使核心证据链失效的未解决问题。 |
| 降级策略 | 在工具、权限、时间或条件不足时，将输出降为当前最佳可核实结果的处理方式。 |
| 结论确定度 | 对当前结论强度的分级表达，通常使用 High、Medium、Low 三档。 |

## 附录 B 快速参考卡

- 用途: 提供问题拆解、搜索、验证、反思、停止条件与降级的速查卡。

<!-- appendix:b:start -->
### 最小行动卡

#### 1. 启动与执行

- 启动条件: 问题复杂、信息冲突、结论高利害，且需要同时交付证据链、推理链和确定度。
- 进入前先写清楚: 研究对象、主体/时间/地点/版本边界、本轮任务目标、不纳入范围。
- 问题拆解最小模板: 主问题、子问题、主假设、替代假设、信息需求。
- 搜索切换: 先广搜看版图，再深搜补断点；若新材料只是重复转述，就退出当前路径。
- 每轮动作后: 记录新增证据、更新判断变化、给出下一步唯一优先动作。

#### 2. 收束与交付

- 证据底线: 关键结论不能只靠单一来源；共用同一上游的来源不算独立验证；关键定义、口径、版本问题优先回原文。
- 停止条件: 核心命题已被多源支撑，或连续几轮没有新增独立证据，或外部条件已阻断继续推进。
- 降级交付: 冻结已确认的证据，下调结论确定度，说明失败环节，并给出最现实的补救路径。
- 输出前三问: 关键结论能否逐条追到证据；最可能推翻它的反证是否检验过；另一个无状态 LLM 能否复现你的路径。

## 附录 C 引用与证据使用约定

- 用途: 若后续决定保留真实引文体系，在此统一说明引用格式与证据标注规则。

<!-- appendix:c:start -->
本技能采用无伪引文（clean, no fake citation）的证据使用约定。凡是进入研究输出的证据，都应满足“能指出来源、能说明作用、能交代限制”这三个最低条件。

### 最小证据规则

- 写明来源类型与来源位置；若有真实链接、文档名、数据库名、发布日期、版本号，应尽可能保留。
- 写明时间信息；高时效问题必须交代发布日期、访问时间或版本时间。
- 写明论证作用；每条证据应标记为支撑、反驳、背景或待核。
- 写明使用限制；若未见原件、来源独立性不足、口径不明，必须同时披露。
- 不得伪造编号、脚注、论文名或出处；不得把“常识”“有人说”这类模糊表述当作证据。
- 强结论需要强证据；若仍有致命缺口，结论必须条件化表达，并把缺口写入残余不确定。

### 最小合规模板

可以用下面这组字段记录关键证据，哪怕不采用正式脚注体系，也能保持可复核性。

```md
- 证据: <事实内容>
  - 来源: <机构 / 文档 / 页面 / 数据库>
  - 时间: <发布日期 / 访问时间 / 版本>
  - 类型: <原始 / 二手 / 线索>
  - 作用: <支撑 / 反驳 / 背景 / 待核>
  - 限制: <未见原件 / 独立性不足 / 口径待核 / 时效受限>
```

附录 C 的职责不是教格式排版，而是守住研究诚信（research integrity）的底线：没有真实来源，就不要伪造引用；证据还不够，就不要把判断写满；限制条件存在，就必须随结论一起交付。
<!-- appendix:c:end -->
